# ПЛАН: Русская локализация Orwell: Ignorance is Strength

## Контекст

**Проблема:** Игра Orwell: Ignorance is Strength (Steam, Unity 5.6.3) не имеет русской локализации. Существующий "русификатор" от Dual Crew (2018) — это просто подмена конфига, дающий "квадраты" вместо кириллицы. На ZoneOfGames ведётся параллельный проект (Amigaser + Saimys87), но прогресс медленный.

**Цель:** Создать полноценный русификатор с установщиком для бесплатной раздачи. После установки игра **должна стабильно запускаться из Steam без вылетов**.

**Версия игры:** 1.1.9207.35014 (подтверждена в Assembly-CSharp.dll)
**Движок:** Unity 5.6.3 (Build 13832220)
**Steam Build ID:** 17763934

## Архитектура текста в игре (4 слоя)

| Слой | Описание | Объём | Хранение |
|------|----------|-------|----------|
| 1. SmartLocalization | Меню, кнопки, системные сообщения | ~50 ключей | `resources.assets` (ResX XML, en + de) |
| 2. TextMeshPro в .assets | Статические UI-элементы | ~10-30 элементов | `sharedassets0-4.assets` |

**Ключевые находки:**
- Игра использует **SmartLocalization** (не I2Localization) — легковесная XML-based система
- Уже есть немецкий (de) и английский (en) языки
- Нет DRM, нет проверки целостности файлов, нет антитамперинга
- 88 SDF-шрифтов, основной — Ubuntu family, **кириллицы нет**

## ФАЗА 1: Экстракция текстов

### Статус: НЕ НАЧАТА

### 1.1 SmartLocalization (resources.assets)
- Открыть в UABEA → найти ScriptableObject SmartLocalization
- Экспортировать ResX XML с ключами en/de
- Сохранить в `originals/smartloc/en.json`
- ~50 ключей: `INGAMEMENU_*`, `MAINMENU_*`, `PROFILER_*`, `LISTENER_*`, `READER_*`, `BASIC_CALENDAR_*`

### 1.2 TextMeshPro из .assets
```bash
Patcher.exe unpack -i "Ignorance_Data" -c TextMeshProUGUI --group type_source
```
Альтернатива: UABEA → Filter by TextMeshProUGUI → Export Dump
- Целевые файлы: `sharedassets3.assets` (16 MB, ~10 TMP-объектов)
- Сохранить с path_id для обратного импорта

### 1.3 AssetBundles (КРИТИЧЕСКАЯ ЗАДАЧА — 90% контента)
Для каждого из 32 бандлов в `PCIGN/`:
1. UABEA: File > Open AssetBundle
2. Filter by MonoBehaviour (TextMeshPro)
3. Export Dump → JSON с `m_text`
4. Сохранить в `originals/bundles/{bundle_id}_{path_id}.json`

**Важно:** сохранять метаданные (path_id, file_id, bundle_name) для обратного импорта

Автоматизация: скрипт `tools/extract_bundles.py` через AssetsTools.NET

### 1.4 Хардкод-строки из DLL
dnSpyEx → Ctrl+Shift+K → поиск строковых литералов:
- Episode names: "Episode One: Thesis", "Episode Two: Antithesis", "Episode Three: Synthesis"
- Confirmations: "Are you sure you want to load/quit/finish..."
- UI: "Continue playing", "New Objective", "Objective solved"
- System: "YOU ARE USING THE OFFICE VERSION OF ORWELL."
- ~50-100 строк

Сохранить в `originals/dll/hardcoded_strings.json`

### 1.5 Инвентаризация
Скрипт `tools/inventory.py`:
- Подсчёт строк по источникам
- Классификация по контексту (UI, чат, статья, документ, досье)
- Генерация `originals/inventory.csv`

**Контрольная точка:** Полный инвентарь ~5500-6000 строк. Все тексты экспортированы с метаданными для обратного импорта.

## ФАЗА 2: Подготовка шрифтов (параллельно с Фазой 1)

### Статус: НЕ НАЧАТА

### 2.1 Получение кириллических шрифтов
- **Приоритет:** Шрифты от Amigaser (ZoneOfGames, Яндекс.Диск)
- **Резерв:** Генерация через TextMeshPro Font Asset Creator в Unity 5.6.3:
  - Source: Ubuntu-Regular/Bold/Medium/Light.ttf
  - Character Set: ASCII (32-126) + Cyrillic (1024-1279) + пунктуация
  - Atlas: 4096x4096, Render Mode: SDFAA, Padding: 5

### 2.2 Замена шрифтов через UABEA
1. Найти path_id оригинальных Ubuntu SDF Font Assets
2. Import Dump: заменить Atlas (текстура) + таблицу глифов + метрики
3. **Формат атласа ДОЛЖЕН совпадать** (DXT5 или ARGB32)
4. **Метрики максимально близки** к оригиналу (m_LineHeight, m_Ascender, m_Descender)

### 2.3 Проверка шрифтов (без текстов)
1. Заменить только шрифты в копии ассетов
2. Запустить игру
3. Проверить: Latin по-прежнему отображается, кириллица рендерится, нет вылетов

**Контрольная точка:** Игра работает с кириллическими шрифтами.

3. **Менять ТОЛЬКО `m_text`** — не трогать transform, layout, sizes

4. File > Save Module

## Правила стабильности (КРИТИЧНО)

1. **НИКОГДА не модифицировать:** `Ignorance.exe`, `steam_api.dll`, `UnityEngine.dll`, `mscorlib.dll`, `Mono.dll`
2. **DLL-патчинг:** ТОЛЬКО замена ldstr операндов, никаких структурных IL-изменений
3. **AssetBundles:** сохранять с тем же сжатием, менять ТОЛЬКО текстовые поля
4. **Шрифты:** атлас того же формата и близкого размера
5. **Тестировать после КАЖДОГО изменённого файла**
6. **Бэкап ВСЕГДА перед модификацией**
7. **Не ломать Steam-интеграцию** — не трогать steam_api.dll и CSteamworks.dll
